Soru-Cozum Application Api Information

  • There are 4 main api address:

    • ..../api/** : includes all operation excepts signup and login, requires token
    • ..../signup/** : signup request, We plan not to add signup page in the mobile application, we will add all students and teachers by ourselves, doesn't require token
    • ..../login/**: login request, doesn't require token
    • ..../info/** : this page, doesn't require token

Note: Publisher should be already created before any request. Otherwise no one(student) can upload question to the server

Student Request

Ask Question Request POST (localhost:8080/api/student/addQuestionToStudent)

  • Content Type must be multipart form data

  • Each body parameter should be sent via key-value pair.

  • You must specify all the categories, otherwise error will return.

  • Question category is a must .And they are static string values:

    • "Matematik", "Biyoloji", "Geometri", "Fizik", "Kimya"
    • If you don't specify or undefined category will return an error.

  • You must specify Question sub category = "nonce", if not exists . For example if there is question with no sub category then send as "nonce", otherwise error will return. These sub categories will be static in the future. (For example, "Türev", "Hücrelerin Yapısı" vss)

  • Request

  • After sending request, there are 3 options(excepts errors), student can ask the question, or student asked question before or another student asked question before then only add question to the student's list.
  • Response will also include questionImageDownloadJSON which you can use this json to download question.
xxxxxxxxxx

Student All Questions GET (localhost:8080/api/student/getStudentQuestionList/{studentId})

  • Student will get all the question he/she asked.

  • Response structure:

    • JSON list of all questions

    • Each json question contains:

      • questionImageJSON = json file to download image
      • answerImageJSON = json file to download answer image
      • answerAudioJSON = json file to download audio file
